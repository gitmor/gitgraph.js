Notes

  • Feature branches branch off of develop & merge back into develop
  • Release branches branch off of develop & merge back into develop and master
  • Hotfix branches off of master & after critical bug fixes merge back into develop and master

Pros

  • Git flow model used by open source projects and other firms.
  • Has been trending well last few years.
  • origin/master would be the main branch where the source code of HEAD always reflects a production-ready state.
  • origin/develop would be the main branch where source code of HEAD always reflects a state with latest delivered development changes for next release. aka “integration branch” or "nightly build branch"

Cons

  • Requires more effort since it involves maintaining more branches and merges may not be smooth.
  • Might be overkill if code base does not change often or less available resources.
  • Might be problematic if release cycles are long and same mature code base is changed/refactored extensively between releases along with personnel turnover.
